Problem Description

The rendering engine that we worked on for HW3 has a big limitation - it works under the assumption that light will only scatter once it hits a surface. However, this is not the case in certain conditions like fog or haze. In this case, light may scatter before it hits a surface. This can help improve the realism and the capability of the ray tracing engine that we have already worked on. This is more difficult, however, as it requires more computation power and is more complex to implement due to rays having to interact with particles in the space between surfaces.

Goals and Deliverables

There will be two images we aim to render correctly for this project. First, a laser shining in a Cornell box with uniformly distributed fog. Without volumetric scattering, this would just appear as a dot on the wall. With volumetric scattering, we should see the beam of light as it travels across the box. Second, a more realistic rendering of a streetlight on a foggy night. This will have a more widespread light source with nonuniform fog in the scene. We can compare our renders with ones done by online renderers like Mitsuba 3, both to check accuracy of the render and runtimes. We plan to answer the question of how we can update our existing ray tracer to also do volumetric scattering.

If we are able to, we can also implement a system to adjust parameters with the fog and how light interacts with it via a GUI - increasing fog density, adjusting the Russian Roulette parameters, etc. This GUI could render a low-fidelity preview for the user before rendering the full image.

Schedule

Week 1

  • Create a structure to store the properties of the air/fog/smoke/other medium and how it interacts with light (absorption, scattering, etc)
  • Add function to calculate transmittance of light through a medium
  • Combine transmittance with original ray loop of HW3 to get new color after passing through medium

    Week 2

  • Change medium to be non-uniform in density, and write function to query density at a given point
  • Update ray loop to check density's effect on transmittance at each time step
  • Update bounding boxes to increase performance

Week 3

  • Implement phase function to determine how light scatters when it hits a particle
  • Use shadow rays from intersection of rays to light source to attenuate light as it moves through medium
  • Reduce noise by picking new ray directions according to phase function instead of randomly

Week 4

  • Allow rays to bounce off of volume particles
  • Update russian roulette for volumetric rendering
  • Final renders to display engine's capabilities
