Problem Description

The rendering engine from HW3 has a significant limitation: it assumes light only scatters upon hitting a surface. In reality, phenomena like fog, haze, or smoke cause light to scatter before reaching a solid object. Implementing volumetric scattering enhances realism but introduces complexity, as rays must now interact with particles throughout the medium, requiring significantly more computational power.

Goals and Deliverables

Our primary goal is to render two specific scenarios accurately:

  • The Cornell Laser: A laser beam traveling through uniform fog inside a Cornell box. We expect to see the visible path of the beam rather than just a terminal dot on the wall.
  • Foggy Streetlight: A realistic night scene with a wide light source and non-uniform fog density.

We will validate our results against the Mitsuba 3 renderer to compare both visual accuracy and runtime performance. If time permits, we plan to implement a GUI for real-time parameter adjustment (e.g., fog density, Russian Roulette thresholds) with a low-fidelity preview mode.

Schedule

Week 1: Fundamentals

  • Define data structures for medium properties (absorption, scattering, etc.).
  • Implement transmittance calculation functions.
  • Integrate medium transmittance into the core HW3 ray loop.

Week 2: Heterogeneous Media

  • Implement non-uniform density support and density query functions.
  • Update the ray loop to account for density-step variance.
  • Optimize performance using specialized bounding volume hierarchies.

Week 3: Phase Functions & Sampling

  • Implement phase functions (e.g., Henyey-Greenstein) for scattering direction.
  • Integrate shadow rays for light attenuation through the medium.
  • Implement importance sampling based on the phase function to reduce noise.

Week 4: Final Polish

  • Enable multi-bounce volumetric scattering.
  • Refine Russian Roulette logic for volumetric paths.
  • Generate final high-quality renders for the showcase.
